# Devpost Submission Guide

Devpost will be used for final project submissions.

Create or sign in to Devpost here:

[Devpost](https://devpost.com)

The official Devpost event link will be shared by the organizers.

---

## What to Submit

Each team should submit the following on Devpost:

- Project name
- Team name
- Team members
- Short project description
- Problem being solved
- Solution overview
- Tools used
- Demo link, video, or screenshots if available
- Any notes about what is working and what is still in progress

---

## Suggested Project Description

Your project description should answer:

1. What problem are you solving?
2. Who does this help?
3. What did you build?
4. How does it work?
5. Which tools did you use?
6. What is working now?
7. What would you improve with more time?

---

## Tools Used

Please mention which tools your team used, such as:

- [Cursor](https://cursor.com)
- [Lovable](https://lovable.dev)
- [Apify](https://apify.com)
- [Valsea / Valencia API](https://valsea.ai/dashboard/billing)
- Other AI tools
- Other APIs, datasets, or libraries

---

## Demo Tips

Your demo should be short, clear, and focused.

A good demo includes:

- The problem
- The solution
- The working prototype
- The tools used
- The impact or value of the idea

Do not spend too much time explaining everything. Show what works.

---

## Before Submitting

Before submitting on Devpost, check that:

- Your project has a clear name
- All team members are listed
- Your description is easy to understand
- Your demo link or screenshots work
- You mention the tools used
- You do not expose private API keys
- Your submission is completed before the deadline
