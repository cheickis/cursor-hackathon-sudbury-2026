# FAQ

## When is the event?

The event is on Saturday.

## Where do I register?

Register on Luma:

https://luma.com/cursor-sudbury-canada

## Where do I get event updates?

All important updates will be shared through the Discord channel.

Please join the Discord before the event:

https://discord.gg/Px5hHgSCt

## Do I need to join Discord?

Yes. Participants should join the Discord channel for:

- Event announcements
- Schedule updates
- Team formation
- Tool setup help
- Devpost submission reminders
- Questions during the hackathon

Discord invite:

https://discord.gg/Px5hHgSCt

## Do I need a GitHub account?

No. Participants do not need to create a GitHub account.

The event will use Devpost for project submissions.

## Do I need a Devpost account?

Yes. Devpost will be used for final project submission.

Go to:

https://devpost.com

The official event submission link will be shared by the organizers.

## Do I need Cursor installed?

Yes. Please install Cursor before the event:

https://cursor.com/downloads

## Do I need a Cursor account?

Yes. Please create a Cursor account before the event:

https://cursor.com

## Do I need to know how to code?

You do not need to be an expert. Cursor and the other tools can help you build, debug, and explain code.

Teams are encouraged to work together.

## Can I use AI tools?

Yes. Participants are encouraged to use Cursor and other AI tools.

Make sure you understand and test the code you use.

## Can I use Lovable?

Yes. Lovable can help you build a web app or prototype quickly.

## How do I activate Lovable credits?

Go to the [Credits and API Access](credits.md) page and follow the Lovable instructions.

The event discount code will be shared with registered participants during the event.

## Can I share the Lovable discount code?

No. The code is only for registered participants and should not be shared outside the event.

## Can I use Apify?

Yes. Apify can be used for web scraping, automation, data collection, and AI workflows.

## How do I activate Apify credits?

Go to the [Credits and API Access](credits.md) page and follow the Apify instructions.

If an event code is available, it will be shared with registered participants.

## Can I use Valsea / Valencia API?

Yes. Follow the instructions on the [Credits and API Access](credits.md) page.

## What does the Valsea / Valencia Free plan include?

The Free plan includes **55 minutes one-time**.

The plan has a hard cap, so requests stop once credits are exhausted.

## Should I share my API keys?

No. Never share API keys publicly.

Keep API keys private and store them safely in your own project.

## What should my team submit?

Submit your project on Devpost.

At minimum, your team should have:

- Project name
- Team members
- Short project description
- Working demo or prototype
- Demo link, video, or screenshots if available
- List of tools used

## What should the project description include?

Your project description should include:

- Project name
- Team members
- Problem
- Solution
- Tools used
- What is working
- Known limitations
- What you would improve with more time

## Can I work alone?

Yes, unless the event organizers announce a different team rule.

## Can I join a team at the event?

Yes. Participants can form teams during the event.

## Can I start with an existing idea?

Yes. You can bring an idea, but the main building should happen during the hackathon.

## Who do I ask for help?

Check this FAQ first.

If your question is not answered here, ask in the Discord channel or speak with one of the organizers during the event.
