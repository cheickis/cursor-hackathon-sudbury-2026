# Tool Setup Guide

This page explains how to set up the main tools for the hackathon.

---

## 1. Cursor

Cursor is the AI code editor we will use during the hackathon.

### Create a Cursor Account

1. Go to https://cursor.com
2. Click **Sign in** or **Get started**.
3. Create an account using your email, Google account, or another available sign-in option.
4. Download Cursor: https://cursor.com/downloads
5. Install Cursor.
6. Open Cursor and sign in.

### Start a Project in Cursor

You can start in two ways.

### Option A: Create a New Local Project

1. Create a new folder on your computer.
2. Open Cursor.
3. Open the folder in Cursor.
4. Start creating your project files.

Recommended files:

- README file
- Main app file
- Frontend or backend folder if needed
- Notes for your team

### Option B: Use an Existing Project Folder

1. Open Cursor.
2. Open your existing project folder.
3. Ask Cursor to help you understand, improve, or debug the project.
4. Continue building from there.

### How to Use Cursor During the Hackathon

Use Cursor to help you:

- Generate code
- Explain errors
- Create a project structure
- Build frontend pages
- Build backend APIs
- Connect APIs
- Write documentation
- Debug issues
- Improve your demo

### Good Cursor Prompts

Try prompts like:

- Create a simple project structure for this hackathon idea.
- Explain this error and suggest a fix.
- Help me connect this API key safely.
- Create a short project description for Devpost.
- Review this project and tell me what could break during the demo.
- Help me prepare a short demo script.

Important: do not blindly accept AI-generated code. Read it, test it, and make sure you understand what it does.

## 2. Devpost

Devpost will be used for final project submission.

Participants should use Devpost to submit:

- Project name
- Team members
- Short project description
- Demo link, video, or screenshots if available
- Tools used
- What was built during the hackathon
- Any additional resources required by the organizers

The official Devpost event link will be shared by the organizers.

More details are available in the [Devpost Submission Guide](devpost-submission.md).

## 3. Lovable

Lovable can help you quickly create web apps, dashboards, and prototypes.

1. Go to https://lovable.dev
2. Create an account or sign in.
3. Start a new project.
4. Describe what you want to build.
5. Use Lovable to generate, edit, and improve your app.

Lovable is useful for:

- Landing pages
- Dashboards
- Demo apps
- Internal tools
- Quick prototypes

---

## 4. Apify

Apify can be used for web scraping, automation, data collection, and agent workflows.

1. Go to https://apify.com
2. Create an account or sign in.
3. Open the Apify Console.
4. Create or use an Actor.
5. Get your API token from account settings.
6. Store the token safely in your project.

Apify is useful for:

- Collecting web data
- Automating browser workflows
- Monitoring websites
- Building data pipelines
- Feeding data into AI apps

---

## 5. Valsea / Valencia API

Use Valsea / Valencia API for voice, audio, or related AI features.

1. Go to https://valsea.ai/dashboard/billing
2. Sign in or create an account.
3. Choose the Free plan.
4. Go to **API Keys** in the sidebar.
5. Create or copy your API key.
6. Store the key safely in your project.

Valsea / Valencia API is useful for:

- Voice features
- Audio interaction
- Speech-based applications
- AI-powered communication tools

---

## API Key Safety

Do not paste private API keys into public documents, screenshots, or shared chats.

Keep API keys private and only use them inside your own project environment.
